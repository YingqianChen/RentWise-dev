# RentWise 项目概述

> 本文档为团队提供项目的整体介绍，帮助理解项目的目标、功能和价值。

---

## 1. 问题陈述

### 香港租房市场的痛点

在香港租房市场，租客面临诸多挑战：

| 痛点 | 描述 |
|------|------|
| **信息过载** | 大量房源信息，难以系统化比较 |
| **隐性成本** | 除了租金，还有管理费、差饷、押金等多种费用 |
| **条款复杂** | 租约条款繁杂，普通人难以理解其中的风险 |
| **决策困难** | 面对多个候选房源，不知道如何做出最优选择 |
| **语言障碍** | 部分房源信息为中英混合，理解困难 |

### 典型用户场景

> *"我有几个租房选择，但不知道下一步该怎么办..."*

这是许多租客的真实困惑。传统的解决方案往往是：

- 随意选择一个"看起来不错"的房源
- 依赖中介的建议（可能存在利益冲突）
- 花费大量时间手动记录和比较

---

## 2. 解决方案

### RentWise 的定位

**RentWise 是一个 AI 驱动的香港租房决策助手**，专注于帮助用户：

```
收集房源 → 分析评估 → 对比决策 → 行动建议
```

### 核心理念

RentWise 不是一个简单的"房源提取器"或"聊天机器人"，而是：

- **Candidate-pool decision workspace** — 候选池决策工作台
- **Compare-driven shortlist tool** — 对比驱动的精选工具
- **Agent-assisted explanation layer** — 智能助手解释层

### 与传统方法的对比

| 传统方法 | RentWise |
|----------|----------|
| 手动记录房源信息 | 自动提取和结构化 |
| 主观判断好坏 | AI 辅助评估风险 |
| 难以对比多个房源 | 智能对比和分组 |
| 不知道下一步做什么 | 明确的行动建议 |

## 3. 核心功能一览

### Phase 1: 候选池工作流 ✅ 已完成

```
用户注册 → 创建搜索项目 → 导入房源候选 → AI 分析评估 → 仪表盘查看 → 做出决策
```

#### 3.1 用户管理

- 用户注册与登录
- JWT 令牌认证
- 个人项目管理

#### 3.2 搜索项目 (Search Project)

- 创建租房搜索项目
- 设置预算范围
- 定义偏好区域
- 指定必须条件和排除条件

#### 3.3 候选房源导入 (Candidate Import)

- **文本输入**：直接粘贴房源描述、聊天记录
- **图片上传**：支持多张图片，自动 OCR 文字识别
- **混合输入**：文字 + 图片组合
- **后台处理**：异步处理，不阻塞用户操作

#### 3.4 AI 分析评估

- **信息提取 (Extraction)**：从非结构化文本提取租金、面积、位置等信息
- **成本评估 (Cost Assessment)**：分析总成本与预算匹配度
- **条款评估 (Clause Assessment)**：识别租约条款中的风险点
- **整体评估 (Candidate Assessment)**：综合评估并给出推荐

#### 3.5 仪表盘 (Dashboard)

- **优先级队列**：显示需要优先关注的房源
- **调查清单**：列出需要用户跟进的问题
- **统计数据**：候选池状态概览
- **AI 建议**：当前最优先的行动建议

#### 3.6 候选详情 (Candidate Detail)

- 查看完整提取信息
- 查看各项评估结果
- 查看基准数据（如适用）
- 执行操作：短名单、拒绝、重新评估、编辑、删除

---

### Phase 2: 对比工作流 ✅ 已完成

#### 3.7 候选对比 (Compare)

- 从仪表盘选择多个候选房源
- AI 生成分组：
  - **Best current option** — 当前最佳选择
  - **Viable alternatives** — 可行的备选方案
  - **Not ready for comparison** — 信息不足，暂无法公平比较
  - **Likely drop** — 建议放弃
- 对比卡片解释每个房源的优劣
- **AI Agent Briefing**：智能决策简报
  - Current take — 当前判断
  - Why now — 为什么现在是决策时机
  - What could change — 什么因素可能改变判断
  - Today's move — 今天的行动建议

## 5. 关键概念解释

### 5.1 Search Project（搜索项目）

用户创建的租房搜索任务，包含：

- 预算范围
- 偏好区域
- 必须条件（如：必须允许养宠物）
- 排除条件（如：不接受顶楼）

**类比**：就像一个"租房文件夹"，用来组织同一次找房的所有候选房源。

### 5.2 Candidate（候选房源）

用户导入的每一个租房选项，包含：

- 原始文本（房源描述、聊天记录等）
- 上传的图片
- AI 提取的结构化信息
- 各项评估结果

**状态流转**：

```
new → needs_info → follow_up → high_risk_pending → shortlisted/rejected
```

### 5.3 Assessment（评估）

AI 对候选房源的分析结果：

| 评估类型 | 内容 |
|----------|------|
| **Cost Assessment** | 租金、押金、管理费等成本分析，与预算匹配度 |
| **Clause Assessment** | 租约条款风险识别（如：退租条款、维修责任） |
| **Candidate Assessment** | 综合评估，给出推荐级别和下一步行动 |

### 5.4 Investigation Item（调查项）

AI 识别出的需要用户跟进确认的问题。

**例子**：

- "是否包含家具？"
- "实际使用面积是多少？"
- "水电费如何计算？"

**仪表盘会将相似问题分组**，避免重复显示。

### 5.5 Compare（对比）

将多个候选房源放在一起比较，AI 提供：

- 分组建议（最佳、备选、信息不足、建议放弃）
- 每个房源的优劣分析
- 明确的行动建议

## 6. 项目开发背景

### 技术实现方式

本项目采用 **Vibe Coding**（AI 辅助编程）方式开发：

- 通过与 AI 对话，逐步构建功能
- AI 帮助生成代码、设计架构、解决问题
- 团队成员专注于产品逻辑和业务理解

### 项目演进

```
Streamlit 原型 → FastAPI + Next.js 单体仓库 → 当前版本
```

- **legacy/** 目录保留了早期的 Streamlit 原型代码
- 当前版本采用现代前后端分离架构

## 7. 项目价值总结

### 对用户的价值

- **节省时间**：自动整理和比较房源信息
- **降低风险**：识别潜在的成本陷阱和条款风险
- **明确行动**：知道下一步该做什么

### 技术价值

- 展示了 AI 在实际业务场景中的应用
- 结合了 OCR、LLM、Agent Workflow 等多种 AI 技术
- 实现了从"信息提取"到"决策支持"的完整链路
